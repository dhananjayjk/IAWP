Need Setting Up Replica Set MongoDB

We develop many applications with various database support at backend. But standalone databases can be single point of failure in case of very large systems. We can't afford failure of system but to ensure that we have to invest in data replica which serves in case of failure of any nodes(Servers). To support read-heavy applications like instagram, twitter secondary replicas can be used for read queries only. Write queries can be served by primary database nodes. Once write is successful in a primary node, it is propagated throughout replicaset. oplog.rs in MongoDB does the same work by logging all transactions on primary. In case of failure of primary database server, secondary replica set having most recent data can be elected as primary. Sharding practices are used as divide & conquer stratergy but it is advisable to use replica to ensure safe & consistent delivery of dat. #ReplicateData&StayCalm
